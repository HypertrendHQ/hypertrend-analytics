# Contributing to HyperTrend Analytics

感谢您对 HyperTrend Analytics 的兴趣！我们欢迎各种形式的贡献。

## 🚀 如何贡献

### 报告 Bug

如果您发现了 Bug，请提交 Issue，并包含以下信息：
- 问题描述
- 复现步骤
- 预期行为
- 实际行为
- 环境信息（OpenClaw 版本、操作系统等）

### 提交新功能

1. **Fork** 本仓库
2. 创建您的 **Feature Branch** (`git checkout -b feature/amazing-feature`)
3. **Commit** 您的更改 (`git commit -m 'feat: add amazing feature'`)
4. **Push** 到 Branch (`git push origin feature/amazing-feature`)
5. 提交 **Pull Request**

### 改进文档

文档改进也是非常受欢迎的贡献！包括：
- 修正拼写错误
- 改进示例
- 添加更多使用场景
- 翻译文档

## 📝 Commit 规范

我们使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建/工具相关

示例：
```
feat: add whale position history tracking

- Add 7d/30d position change analysis
- Include PnL trend charts
- Update examples with historical data
```

## 🎨 代码风格

- 使用清晰的命名
- 添加必要的注释
- 保持代码简洁
- 遵循现有代码结构

## 📋 Pull Request 流程

1. 确保您的 PR 有清晰的标题和描述
2. 关联相关的 Issue（如果有）
3. 确保所有 CI 检查通过
4. 等待维护者审核
